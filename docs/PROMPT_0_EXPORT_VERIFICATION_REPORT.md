## 10. TrialReadiness / diagnostics logic (pre-export checks)

**Location:** `lib/features/diagnostics/trial_readiness_service.dart` — `TrialReadinessService.runChecks(String trialId, Ref ref)`.

**Checks performed (in order):**

1. Trial exists (getTrialById); else blocker "Trial not found".  
2. Plots defined (getPlotsForTrial); empty → blocker "No plots defined".  
3. Treatments defined (getTreatmentsForTrial); empty → blocker "No treatments defined".  
4. Assessments defined (assessmentsForTrialProvider); empty → blocker "No assessments defined".  
5. Assignments present (getForTrial); no assignment with treatmentId → blocker "No plots have treatment assignments".  
6. Sessions exist (getSessionsForTrial); empty → blocker "No rating sessions recorded".  
7. At least one rating across sessions (getCurrentRatingsForSession); totalRatings == 0 → blocker "No ratings recorded".  
8. Seeding event (getSeedingEventForTrial); null → warning "Seeding event not recorded".  
9. Application events (getApplicationsForTrial); empty → warning "No application events recorded".  
10. Sessions before first application date → warning "Session recorded before first application" (with session names).  
11. Unrated plots (ratedPlotsCountForTrialProvider vs plot count); unratedCount > 0 → warning "$n plots have no ratings".  
12. Treatments without components (getComponentsForTreatment); any empty → warning "One or more treatments have no components".  
13. Assessments defined in session but never used in ratings → warning "Assessment defined but never used: $name".  
14. Application events with missing product or rate → warning "Application event has incomplete details".  

**Result type:** `TrialReadinessReport` with list of `TrialReadinessCheck` (code, label, detail, severity). Severity: blocker / warning / pass. Export is gated by blockers; warnings can show "Export anyway".
